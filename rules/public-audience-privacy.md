# Public Audience Privacy

- Before writing or modifying any artifact, identify the intended audience: private, internal, client-facing, public, reusable, or unknown.
- Treat unknown or reusable audiences as public by default.
- For public, reusable, external, or uncertain audiences, use synthetic examples and generic placeholders.
- Do not include real person names, client names, internal organization names, private domain names, absolute local paths, hostnames, IP addresses, account identifiers, emails, private URLs, ticket IDs, screenshots, copied logs, transcripts, or environment-specific details.
- If a real internal detail seems necessary, stop and ask for explicit approval before including it.
- Apply this to repositories, documents, README files, examples, templates, comments, fixtures, changelogs, PR text, reports, prompts, screenshots, and generated artifacts.
