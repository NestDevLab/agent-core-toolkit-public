# Write Docs Skill

Use this skill when creating or updating documentation that future users or agents will rely on.

## Documentation Standards

- Identify the canonical source before writing.
- Prefer durable guidance over chat-history summaries.
- Keep examples generic unless the target document is explicitly private.
- Avoid duplicating volatile details when a pointer is enough.

## Process

1. Determine the audience and the action the document should enable.
2. Read nearby documentation to match tone and structure.
3. Write concise sections with clear headings.
4. Include validation, ownership, or update guidance where relevant.
5. Remove stale or conflicting wording when replacing guidance.
